Project details:
- We are writing a C program that "hex dumps" an attached file.
- The attached file is output in the following format:
  - The number of bytes in the file, in hex (i.e. the memory address)
  - 16 consecutive bytes, in hex, separated by spaces
  - ASCII representation of the bytes, with . replacing non-ASCII bytes

    Fig. 1: Example output of the program

    00002040 75 6e 74 75 20 31 33 2e 33 2e 30 2d 36 75 62 75 |untu 13.3.0-6ubu|
    00002050 6e 74 75 32 7e 32 34 2e 30 34 2e 31 29 20 31 33 |ntu2~24.04.1) 13|

- Additionally, we are compiling said program with optimization options -O0, -O1 and -O3.
- We will analyze and discuss the different ASM code generated by different optimization levels.
